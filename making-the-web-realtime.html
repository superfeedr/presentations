PubSubHubbub

  • A simple, topic(feed)-based publish/subscribe protocol
  • A single API for web-scale, low-latency messaging
  • Three participants: Publisher, Subscriber, Hub
  • Pragmatic (i.e., not theoretically perfect, but solve huge, known use cases with minimal effort for everybody)

5 steps to Epiphany

  1. Publisher selects a hub, and advertises it
    2. <atom:link rel="hub"  href="http://my-hub.superfeedr.com/" />
  2. Subscriber asks the publisher for a resource and finds the hub
  3. Subscriber subscribes to resource on the hub
  4. Publishers updates his content and tells the hub
  5. Hub gets data and pushes it to all subscribers
